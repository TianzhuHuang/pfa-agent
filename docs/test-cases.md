## 1. 认证模块 (Auth)

### TC-1.1 注册流程
| ID | 场景 | 步骤 | 预期结果 | 优先级 |
|---|---|---|---|---|
| 1.1.1 | 正常注册 | 输入有效邮箱+密码(≥6位)+确认密码 → 点击注册 | 注册成功，自动登录，跳转到空状态引导页 | P0 |
| 1.1.2 | 密码太短 | 输入密码 < 6 位 → 注册 | 提示"密码至少 6 位" | P1 |
| 1.1.3 | 密码不一致 | 两次密码不同 → 注册 | 提示"两次密码不一致" | P1 |
| 1.1.4 | 重复注册 | 用已注册邮箱再次注册 | 提示"该邮箱已注册，请直接登录" | P1 |
| 1.1.5 | 空邮箱 | 不填邮箱直接注册 | 提示"请填写邮箱和密码" | P1 |

### TC-1.2 登录流程
| ID | 场景 | 步骤 | 预期结果 | 优先级 |
|---|---|---|---|---|
| 1.2.1 | 正常登录 | 输入正确邮箱+密码 → 登录 | 登录成功，显示 Portfolio Hub | P0 |

| 1.2.5 | 退出登录 | 点击顶部导航栏"退出" | 清除会话，返回登录页 | P1 |

## PM vs QA 冲突点

| 编号 | PM 意见 | QA 意见 | 建议方案 |
|---|---|---|---|
| C1 | 登录持久化应"永久"有效 | 安全角度 refresh_token 应有过期时间 | **决策: 永久** — localStorage 不设过期 |
| C2 | 新用户引导应有动画过渡 | 动画增加复杂度和测试成本 | **决策: 现在做** — fadeInUp + 步骤指示器 |
| C3 | 截图OCR 应支持多张图片批量 | 多图增加 API 成本和超时风险 | **决策: 单张** — 识别后可编辑确认再导入 |

**所有冲突已由产品负责人决策。**
